01 · Project Overview
The Pharma AI Platform is a scientist-facing web application that uses multiple specialized AI/ML models to
automate and accelerate the four core pillars of modern pharmaceutical research. Each feature is backed by a
purpose-trained or fine-tuned model, free public datasets, and runs on Google Colab Pro for accessible GPU
compute.

05 · Fine-Tuning — Step-by-Step for Each Model

Fine-Tuning PRECIOUS3-GPT (Drug & Target Discovery)
Use LoRA (Low-Rank Adaptation) — this is the most efficient method for fine-tuning large models on Colab. LoRA
freezes most weights and trains only small adapter matrices, reducing memory from ~12GB to ~4GB.

Step 1 — Load model with 4-bit quantization
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig import
torch bnb_config = BitsAndBytesConfig( load_in_4bit=True, bnb_4bit_quant_type='nf4',
bnb_4bit_compute_dtype=torch.bfloat16 ) model = AutoModelForCausalLM.from_pretrained(
'insilicomedicine/precious3-gpt-multi-modal', quantization_config=bnb_config,
device_map='auto' ) tokenizer =
AutoTokenizer.from_pretrained('insilicomedicine/precious3-gpt-multi-modal')

Step 2 — Apply LoRA Configuration
from peft import LoraConfig, get_peft_model, TaskType lora_config = LoraConfig( r=16, #
Rank — higher = more expressive but more memory lora_alpha=32, # Scaling factor
target_modules=['q_proj', 'v_proj'], # Which layers to adapt lora_dropout=0.05,
bias='none', task_type=TaskType.CAUSAL_LM ) model = get_peft_model(model, lora_config)
model.print_trainable_parameters()

Step 3 — Prepare Dataset (ChEMBL drug data example)
from datasets import load_dataset # Example: load SMILES + bioactivity labels dataset =
load_dataset('csv', data_files={'train': 'chembl_train.csv'}) def tokenize(example):
return tokenizer(example['smiles'], truncation=True, max_length=512) tokenized =
dataset.map(tokenize, batched=True)

Step 4 — Train with HuggingFace Trainer
from transformers import TrainingArguments, Trainer training_args = TrainingArguments(
output_dir='/content/drive/MyDrive/pharma_ai/precious3_ft', num_train_epochs=3,
per_device_train_batch_size=4, # Keep low for Colab gradient_accumulation_steps=8, #
Simulates batch_size=32 learning_rate=2e-4, fp16=True, # Mixed precision save_steps=500,
logging_steps=100, ) trainer = Trainer( model=model, args=training_args,
train_dataset=tokenized['train'], ) trainer.train()
■ Expected Time on A100 (Colab Pro+): ~2-4 hours for 3 epochs on 50k ChEMBL compounds

Fine-Tuning ChemBERTa (Formulation & Drug Discovery)
ChemBERTa is a BERT model — fine-tune it as a classifier/regressor on molecular property data (solubility,
toxicity). This is straightforward and very fast on Colab.
from transformers import AutoModelForSequenceClassification, AutoTokenizer model =
AutoModelForSequenceClassification.from_pretrained( 'seyonec/ChemBERTa-zinc-base-v1',
num_labels=1 # Regression task (e.g., logS solubility) ) # Your dataset: CSV with 'smiles'
column + 'solubility' column # Tokenize SMILES strings the same way as text # Use MSELoss
for regression, CrossEntropyLoss for classification (toxicity) # Fine-tune same way as
above with Trainer # No LoRA needed — ChemBERTa is small enough to full fine-tune on Colab
■ Expected Time on T4 (free Colab): ~30-45 minutes for ESOL solubility dataset

Fine-Tuning ClinicalBERT (Clinical Trial Analysis)
Use ClinicalBERT for text classification on trial outcome data. Input: trial description text. Output: outcome class
(success/failure/ongoing).
model = AutoModelForSequenceClassification.from_pretrained(

) # Dataset from ClinicalTrials.gov API — extract 'brief_summary' + outcome label #

■■ Important Notes
• License: PRECIOUS3-GPT uses CC-BY-NC-4.0 — free for research and academic use only. If you commercialize
the platform, switch to a commercial-friendly alternative like GPT-J or Mistral fine-tuned on pharma data.
• Colab Disconnect: Colab disconnects after ~12 hours. Always save checkpoints to Google Drive every 500 steps
using save_steps=500 in TrainingArguments.
• Colab Pro Compute Units: Heavy training (PRECIOUS3-GPT) will consume compute units quickly on A100. Budget
~5-10 compute units per training session. Buy extra if needed.
• MIMIC-III Dataset: Requires free registration and credentialing at PhysioNet. Takes 1-2 days for approval. Apply
early.
• Data Size: Start fine-tuning with a 10% sample of large datasets (ChEMBL has 2M+ compounds). A 50k-100k subset
is enough to get strong results.
• RDKit for Visualization: Use RDKit in Python to draw 2D molecule structures. For 3D visualization in the web
frontend, use 3Dmol.js (JavaScript library, free).
